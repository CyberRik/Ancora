# Architecture Decision Records (ADRs)

ADRs capture **narrow, dated** decisions made during implementation — the small forks
in the road that don't warrant a full RFC but that future contributors will ask "why?"
about. RFCs describe the *design*; ADRs record the *decisions* taken while realizing it.

- Use RFCs (`docs/RFC-*`) for subsystem designs.
- Use ADRs for focused choices (e.g. "why asyncpg over psycopg", "why Redis Streams over pub/sub").

Copy [`0000-template.md`](0000-template.md), increment the number, and open a PR.
